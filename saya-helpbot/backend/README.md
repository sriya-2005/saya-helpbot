# SAYA HelpBot — Backend (FastAPI)

## Setup

```bash
cd backend
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Run

```bash
uvicorn main:app --reload --port 8000
```

- API root: http://localhost:8000
- Interactive docs (auto-generated by FastAPI, try requests right in the browser): http://localhost:8000/docs

## Add your real data (no code changes needed)

1. **FAQ**: open `faq.json` and replace the empty `[]` with your 50-100
   entries, matching the shape shown in `faq.example.json`:
   ```json
   [
     { "question": "...", "answer": "..." },
     { "question": "...", "answer": "..." }
   ]
   ```
2. **PDF manuals**: copy your PDF files directly into `docs/`.
3. Either restart the server, or call `POST /reload` to pick up the changes live.

## Endpoints

| Method | Path                     | Purpose                                                |
|--------|--------------------------|---------------------------------------------------------|
| POST   | `/ask`                   | Ask a question. Body: `{"question": "..."}`             |
| GET    | `/suggested-questions`   | All FAQ questions, for the frontend to display as chips |
| POST   | `/reload`                | Re-reads faq.json + docs/ from disk without a restart   |
| GET    | `/health`                | Liveness check                                          |

### Example: `POST /ask`

Request:
```json
{ "question": "how do i reset my password" }
```

Response:
```json
{
  "answer": "1. Go to the login screen.\n2. Click \"Forgot password\"...",
  "source": "FAQ",
  "confidence": 91.0,
  "related_questions": ["What is ReconX?", "How do I create a reconciliation?"],
  "source_document": null,
  "source_page": null
}
```

If the FAQ has no confident match, the same endpoint automatically falls
back to the PDF manuals in `docs/`, and the response looks like:
```json
{
  "answer": "To configure matching rules, open Settings > Matching and select...",
  "source": "PDF",
  "confidence": 58.0,
  "related_questions": ["What is ReconX?"],
  "source_document": "reconx_manual.pdf",
  "source_page": 12
}
```

If neither source has anything confident, `source` will be `"NONE"` and
`answer` will plainly say so — the API never invents an answer.

## Connecting your existing React frontend

Point your existing fetch/axios calls at `http://localhost:8000`. For example,
wherever your frontend currently calls a mock or placeholder function:

```javascript
async function askQuestion(question) {
  const res = await fetch("http://localhost:8000/ask", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ question }),
  });
  if (!res.ok) throw new Error("Request failed");
  return res.json(); // { answer, source, confidence, related_questions, source_document, source_page }
}

async function getSuggestedQuestions() {
  const res = await fetch("http://localhost:8000/suggested-questions");
  return res.json(); // { questions: [...] }
}
```

If your frontend runs on a port other than `5173` or `3000`, add it to
`ALLOWED_ORIGINS` (comma-separated) before starting the server:

```bash
ALLOWED_ORIGINS=http://localhost:5174 uvicorn main:app --reload --port 8000
```

## Tuning match sensitivity

Two constants at the top of `search.py` control behavior:

- `FAQ_CONFIDENCE_THRESHOLD` (default `65.0`) — how confident a FAQ match
  must be before it's returned directly. Lower = more lenient/forgiving
  matching, but higher risk of returning a slightly-wrong FAQ answer.
- `PDF_CONFIDENCE_FLOOR` (default `35.0`) — the minimum score for a PDF
  section to be shown at all, instead of returning `"NONE"`.

Adjust these once you see how real questions score against your real data.
